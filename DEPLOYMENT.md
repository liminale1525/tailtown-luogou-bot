# 自动归档 Bot 上线流程

这份说明用于把本地测试通过的 Bot，安全地放到尾巴镇正式服务器和 VPS 上长期运行。

## 阶段一：本地测试

目标：确认面板和白名单功能稳定，不测试真实大量归档。

1. 启动 Bot。
2. 确认窗口出现：

```text
归档面板版本：single-panel-2026-06-19-9
```

3. 在测试服务器输入：

```text
/归档
```

4. 测试这些功能：

- 启用自动归档
- 关闭自动归档
- 切换归档周期
- 切换检查间隔
- 添加帖子保护
- 查看白名单
- 从白名单移除帖子
- 刷新状态

## 阶段二：尾巴镇试运行

目标：先接入正式服务器，但不让它立刻归档。

1. 把 Bot 邀请进尾巴镇正式服务器。
2. 重新填写 `.env` 里的正式服务器 ID。
3. 运行：

```powershell
.\scripts\register-commands.bat
```

4. 启动 Bot。
5. 在尾巴镇输入：

```text
/归档
```

6. 确认面板能正常显示活跃帖数量。
7. 暂时保持“自动归档关闭”。
8. 先添加几个必须保护的帖子到白名单。

## 阶段三：小范围启用

目标：低风险验证真实自动归档逻辑。

推荐初始设置：

- 周期：30 天
- 间隔：24 小时
- 自动归档：启用

观察 1-2 天：

- 是否有误归档
- 白名单是否有效
- 归档记录是否符合预期
- 管理员是否能理解面板状态

稳定后再改成更积极的设置：

- 周期：15 天或 7 天
- 间隔：12 小时、6 小时或 3 小时

## 阶段四：VPS 长期运行

VPS 可以理解为一台 24 小时在线的远程电脑。Bot 要长期在线，最终应该放在 VPS 上运行。

推荐部署方式：

1. VPS 安装 Node.js。
2. 从 GitHub 下载 Bot 代码。
3. 在 VPS 上创建 `.env`。
4. 安装依赖。
5. 注册指令。
6. 用 `pm2` 启动 Bot，让它掉线后自动重启。

常用命令示例：

```bash
npm install
npm run register
npm install -g pm2
pm2 start src/index.js --name tail-town-archive-bot
pm2 save
```

## 重要安全点

- `.env` 不能上传 GitHub。
- Bot Token 不要发给任何人。
- 正式启用前先确认白名单。
- 正式启用前先保持自动归档关闭。
- 如果出现异常，第一时间点击“关闭自动归档”。
